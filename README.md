# Список исправленных ошибок в соответствии со стандартом Epic Games C++ Coding Style
## Вариант 1
### Task 1:
1. Глобальные директивы
   - Убрано using namespace std в глобальной области видимости, добавлен префикс std:: ко всем объектам стандартной библиотеки
2. Структура проекта
   - Программа разделена на модули согласно стандарту Epic Games
   - Создан заголовочный файл Header.h с прототипами методов класса
   - Реализация методов вынесена в отдельный файл Source.cpp
   - В основном файле Task1.cpp оставлена только функция main
   - Получение длины пароля вынесено из main в отдельный статический метод класса (GetLengthByLevel)
3. Именование классов и методов
   - Методы переименованы в PascalCase
   - Примеры: setCharacterSets → SetCharacterSets, generate → Generate, levelDescription → GetLevelDescription, randomChar → GetRandomChar, charVectorToString → VectorToString
4. Именование переменных
   - Все переменные переименованы в PascalCase (с заглавной буквы)
   - Имена переменных сделаны информативными
   - Примеры: rng → RandomGenerator, charPool → CharacterPool, result → Result, remaining → Remaining
5. Инициализация переменных
   - Все переменные инициализированы при объявлении
6. Генерация случайных чисел
   - Заменён rand() на современный <random> с std::mt19937
   - Использование std::uniform_int_distribution для равномерного распределения
7. Форматирование кода
   - Фигурные скобки открываются в новой строке для всех управляющих конструкций
   - Добавлены фигурные скобки для однострочного if
8. Проверка ввода
   - Добавлена функция ClearInput() для очистки потока ввода
   - Добавлена функция GetValidLevel() для получения корректного уровня
   - Добавлена проверка на некорректный ввод
9. Локализация
   - Добавлена поддержка русского языка: setlocale(LC_ALL, "Russian");
10. Инкремент/декремент
    - Заменён постфиксный дикремент на префиксный: Remaining-- → --Remaining
11. Комментарии
    - Все функции прокомментированы (что делают)
    - Комментарии начинаются с заглавной буквы
    - Добавлен пробел после `//` в комментариях
    - Комментарии заканчиваются точкой
    - Коментарии приведены к нормальному виду, сделаны информативными
12. Логика генерации пароля
    - Для корректной генерации пароля определённого уровня добавлена проверка, какие классы входят в набор символов (CharacterPool)
    - Добавляются только символы их доступных классов
    - Исправлена ошибка, при которой для лёгкого уровня добавлялись цифры и символы
